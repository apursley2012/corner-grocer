Rewriting Corner Grocer From C++ to Python

Software design and engineering enhancement

Software Design and Engineering Enhancement

Artifact Description

The artifact I selected for enhancement is a C++ command-line program titled CornerGrocerApp.cpp. The program reads a text file of grocery item purchases and calculates the frequency of each item. It allows users to view the full frequency list, look up individual item counts, and display a histogram of item quantities. The application uses standard file streams and a map to store item frequencies, with user interaction managed through a basic menu system.

While functional, the code was procedural and lacked advanced error handling, modularity, or features beyond the core requirements. For this enhancement, I rewrote the program from scratch in Python, incorporating modern design patterns, improved user feedback, and several structural and functional enhancements.

Why I Chose It

This project was chosen because it demonstrates my ability to refactor existing code into a cleaner, more scalable version using a different programming language. The original C++ artifact served its purpose but had limitations in usability, structure, and error handling. Rewriting it in Python allowed me to adopt modern programming practices and tools while preserving the original objectives and logic.

What Changed

The enhanced version introduces a GroceryFrequency class that encapsulates all core functionality, making the code more readable, testable, and maintainable. I added input validation, exception handling, and improved user messaging. In addition to replicating the original features, I implemented two new options: one to save a backup of the frequency list and another to save a backup of the histogram.

The application now confirms whether each backup was successfully written and displays the full file path where it was saved, features the original version did not include. The menu expanded from four options to six, allowing users to create backups at any point in the process. Overall, this results in a more robust and user-friendly tool that achieves the same goals as the original C++ version but with clearer structure and enhanced features.

Reflection on the Process

Converting this project from C++ to Python was an excellent opportunity to reimagine how a familiar task can be handled with a more modern, readable, and scalable approach. One of the initial challenges was identifying areas where the original C++ code could be improved, particularly in terms of readability, structure, and user experience. I designed a class-based Python program that separates concerns, reduces repetition, and enhances user feedback throughout the application.

I also faced practical challenges related to file writing and user error handling. For example, I needed to ensure that each backup file could be created successfully and that any issues during writing were clearly reported. Displaying the full file path where backups are saved added transparency and user confidence.

Overall, this enhancement allowed me to apply core software design principles and demonstrate fluency in both C++ and Python. More importantly, it shows my ability to take legacy logic and reimplement it in a way that is modern, modular, and user-friendly.

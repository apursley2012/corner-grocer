Algorithms and Data Structures Enhancement

Building on the Python Version

For this enhancement, I used Corner Grocer again. I originally created it as a C++ console program that counted grocery item frequencies from a text file. For the second enhancement, I built on the new Python version by adding sorting and searching features to improve its usability and functionality.

New Features

The Corner Grocer application received three additional features that moved it beyond basic frequency counting:

  • Sorting items alphabetically: Users can find specific items more easily because the list is organized by item name.
  • Sorting items by how often they are sold: Users can see product sales frequency from highest to lowest, which supports inventory management and ordering decisions.
  • Searching for a specific item: Users can determine the exact number of times an item was sold without browsing the entire list.

These standard inventory and sales-tracking features transform the program into a more complete business tool. They help store owners maintain organized data, understand sales patterns, and make decisions using clearer information.

Modular Structure

I created a separate utility file for sorting and searching functions so the code would remain modular and maintainable. The alphabetical function accepts the frequency dictionary and sorts items by name. The frequency-based function sorts dictionary entries by count in descending order. The search function normalizes the user’s input, checks the stored data, and reports the item’s frequency or explains that it was not found.

Keeping these features in reusable functions supports separation of concerns. The main file remains focused on the menu and application flow, while the utility module handles data processing. This makes the features easier to test, extend, and debug without duplicating logic.

Input Handling and Compatibility

The expanded menu keeps the original options and adds the new sorting and search features without removing existing functionality. Validation confirms that menu choices fall within the accepted range and that item names contain valid characters. Prompt messages and error feedback are written clearly so users understand what the program expects and what happened after each action.

In the browser demo, I also improved normalization so capitalization and common singular or plural forms are treated as the same item. This prevents records such as Apple, apple, and apples from being split into separate totals. Partial matching makes search more practical while preserving exact frequency results.

What This Demonstrates

This enhancement demonstrates programming proficiency, modular design, code reusability, data sorting, search functionality, function abstraction, input validation, and preserved backward compatibility. It also shows how I evaluate the tradeoff between writing logic inline and separating it into helper functions. I chose the modular approach because it improves readability and scalability while keeping performance appropriate for the project.
